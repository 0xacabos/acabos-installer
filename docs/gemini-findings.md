# ACABOS Installer - Technical Findings & Fixes

This document summarizes the investigation and resolution of the boot freeze issue identified on April 21, 2026.

## Investigation Overview

The ACABOS installer builds a Debian Trixie AI workstation on encrypted ZFS using ZFSBootMenu (ZBM) as the authoritative bootloader. The investigation focused on the end-to-end ZFS topology, the ZBM EFI bundle generation, and the handover to the target OS with NVIDIA drivers.

## Diagnosis: Boot Freeze Analysis

Based on the provided forensic image and codebase analysis, the "freeze" before the ZFSBootMenu UI was attributed to four critical failure points:

1.  **Console Deadlock (Serial Port):**
    *   **Issue:** The default ZBM command line included `console=ttyS0,115200n8`.
    *   **Impact:** On modern hardware lacking a physical serial port, the kernel can hang indefinitely during serial initialization, especially at high loglevels.
2.  **Missing ZBM Module in Initramfs:**
    *   **Issue:** The dracut configuration for the ZBM bundle (`10-acabos.conf`) included `zfs` but omitted the `zfsbootmenu` module.
    *   **Impact:** The initramfs lacked the ZFSBootMenu UI and logic. It likely fell back to a standard dracut init, which stalls without a `root=` parameter.
3.  **Invalid ZFS Cache:**
    *   **Issue:** The installer was creating an empty `zpool.cache` file and including it in the initramfs.
    *   **Impact:** ZFS can fail to import the encrypted pool if it encounters an invalid/empty cache file instead of performing a full device scan.
4.  **KMS Handoff Failure (Post-ZBM):**
    *   **Issue:** The OS kernel command line was missing `nvidia-drm.modeset=1`.
    *   **Impact:** While ZBM might have eventually loaded, the subsequent boot into the OS would likely have frozen during the transition from the UEFI framebuffer to the NVIDIA driver.

## Applied Fixes

The following changes were implemented to ensure a successful boot flow:

### 1. ZFSBootMenu Logic (`lib/stage_boot_chain.sh`)
*   **Fix:** Added `zfsbootmenu` to `add_dracutmodules` in the ZBM dracut configuration.
*   **Fix:** Populated `zpool.cache` from the live pool state using `zpool set cachefile=...` before generating the EFI bundle.

### 2. Bootloader Configuration (`config/zfsbootmenu-config.yaml`)
*   **Fix:** Removed `console=ttyS0` from the default and bringup command lines.
*   **Fix:** Added `zbm.timeout=5` to the default profile to ensure automatic boot progression.

### 3. Kernel Parameters (`lib/stage_zfs_create.sh`)
*   **Fix:** Updated the `org.zfsbootmenu:commandline` property to include `nvidia-drm.modeset=1`. This ensures the parameter is passed to the target OS during the kexec handover.

### 4. Verification & Probes (`lib/probes.sh`)
*   **Fix:** Enhanced `probe_boot_chain` to verify the presence of the `zfsbootmenu` module within the generated initramfs using `lsinitrd`.

## Recommendations
*   **Secure Boot:** Ensure Secure Boot is disabled, as the custom EFI bundle generated by `generate-zbm` is not signed.
*   **Resume Flow:** Use `./acabos-install --resume` to re-generate the boot artifacts with these fixes applied.
